• The website structure begins by hosting the 'index.html' at route 'itc505/lab7' upon server startup.
  • Upon opening the site, users encounter a list of three stories, offering interactive options, along with an "Addendum" button placed on the right side.
  • Clicking any story navigates users to an input page, prompting for six inputs. Each input is associated with dynamically changing labels based on the selected story.
  • To facilitate this functionality, the game name is passed as a URL parameter from the index to the input page, enabling the retrieval of the game name and subsequent label customization.
  • After collecting user inputs, clicking the "Go Mad" button results in storing all inputs and the game name within a JSON object named "data."
  • The JSON data is shared with 'server.js' using the 'fetch' API. Error handling mechanisms are implemented to manage potential failures in data sharing.
  • 'server.js' utilizes JSON handling via 'express.json()', parsing post requests to retrieve inputs and the game name for further processing.
  • Subsequently, conditional statements are used to segregate stories based on the game name. User inputs are dynamically integrated into the stories using string interpolation.
  • The data is later transmitted to the story page via 'redirectUrl' encoded with 'encodeURIComponent'. The story HTML page decodes the story from the URL parameter using 'decodeURIComponent'.
